Another telecom provider. Hope these guys prepared well enough for the network load...

After a bit of research, we find that this version of Tomcat is vulnerable to a recently published CVE known as Ghostcat which allows for local file inclusion and, remote code execution if the server allows for file upload. There are many POCs for the exploit available on GitHub. ajpShooter is one that allows for file read and eval of jsp code.

The Tomcat documentation provides the standard directory layout which gives us an idea of what files to look for.

We begin by reading the deployment description which is located at /WEB-INF/web.xml and provides a mapping for servlets and paths.

These are Java class files, but we can use a decompiler (e.g. JAD) to retrieve the Java source code.

We see that uploaded files are being placed into /uploads/ but with a filename generated by the generateFileName function. If we can upload a malicious jsp file, guess the filename, and exploit the Ghostcat vulnerability to eval it, we should be able to achieve code execution.

Alternative

crypto (50pts)

This task is alternative.

alternative.q.2020.volgactf.ru:7780/

Solution

When we navigate to the page (in Chromium), we get a ERR_CERT_AUTHORITY_INVALID warning (by Chromium) telling us that the TLS certificate is self signed. If we proceed despite the warning, we see that the page is very bare. Looking through the details of the certificate, we see a field named Certificate Subject Alternative Name which lists the string s0.many.fields.in.certificate.com as a DNS name. It turns out VolgaCTF{s0.many.fields.in.certificate.com} is the flag.

Solution

We notice in the encryptor script that the key being used is the md5 hexdigest of the current time. The description hints that this script was run two days ago (from the time of the CTF start, presumably) so we simply bruteforce the key with this in mind:

from Crypto.Cipher import AES
from base64 import b64decode
import time
from hashlib import md5

enc = b64decode(open('encrypted').read())

for i in range(-24*60*60*3, 0):
    key = md5(str(int(time.time()+i))).digest()
    aes = AES.new(key, AES.MODE_ECB)
    f = aes.decrypt(enc)
    if 'Volga' in f:

Solution

We see that the ElGamal cryptosystem is being used. In order to get the flag, we must successfully solve a challenge set by the server 1000 times. The answer to each challenge is either 0 or 1, but since we must solve 1000 of these challenges, it is not feasible to bruteforce and guess the answers.

We notice that in each challenge, there are two plaintexts being encrypted. We will call plaintexts[0] s0s_0s0 (or s0) and plaintexts[1] s1s_1s1 (or s1). These plaintexts are generated by the findQNR function and findQR function respectively. Then, the server uses ElGamal to encrypt this plaintext and sends the encryption to us. Our goal is to figure out which plaintext was sent.

Before we attempt to find the vulnerability in the challenge, we must first understand some important concepts. Firstly, we need to understand how ElGamal encryption works, and we need to understand what quadratic residues are.

Mathematical Concepts/Notation

Fp\mathbb{F}_pFp denotes the ring of integers modulo ppp. Fp\mathbb{F}_p^*Fp denotes the group of units modulo ppp, that is, the set {1,2,3,,p1}\{1, 2, 3, \ldots, p - 1 \}{1,2,3,,p1} (because ppp is prime).

An element gFpg \in \mathbb{F}_p^*gFp is called a generator of Fp\mathbb{F}_p^*Fp if its powers generate every element of Fp\mathbb{F}_p^*Fp. i.e.

Fp={1,g,g2,g3,,gp2}\mathbb{F}_p^* = \{1, g, g^2, g^3, \ldots, g^{p-2} \}Fp={1,g,g2,g3,,gp2}

If ppp is prime, such an element always exists.

ElGamal Cryptosystem

The ElGamal cryptosystem is a public key cryptosystem whose security is based on the discrete logarithm problem.

Key Generation: Alice chooses a large prime number ppp and a number gFpg \in \mathbb{F}_p^*gFp that is a generator for Fp\mathbb{F}_p^*Fp. She then chooses a secret number xxx and computes y=gx(modp)y = g^x \pmod py=gx(modp). She publishes (g,p,y)(g, p, y)(g,p,y) as her public key.

Encryption: Suppose Bob wants to send Alice a message mmm using Alice's public key. He generates a random number rrr such that 1r<p1 \leq r < p1r<p. He then computes

c1gr(modp)andc2myr(modp)c_1 \equiv g^r \pmod p \text{\hspace{0.2in} and \hspace{0.2in}} c_2 \equiv my^r \pmod pc1gr(modp)andc2myr(modp)

and sends (c1,c2)(c_1, c_2)(c1,c2) as the ciphertext.

Decryption: Decryption isn't involved in this challenge, but it isn't hard to show that Alice can retrieve the plaintext by computing

mc2(c1x)1(modp)m \equiv c_2(c_1^x)^{-1} \pmod pmc2(c1x)1(modp)

Quadratic Residues

Definition (Quadratic Residue): Let ppp be an odd prime and let aaa be a number such that ppp does not divide aaa. If there exists a number ccc such that c2a(modp)c^2 \equiv a \pmod pc2a(modp), then we say that aaa is a quadratic residue modulo ppp, otherwise, we say that aaa is a quadratic nonresidue modulo ppp.

Proposition (Quadratic Residue Properties): Let ppp be an odd prime number.

  • (i) The product of two quadratic residues modulo ppp is a quadratic residue modulo ppp.
  • (ii) The product of a quadratic residue and a quadratic nonresidue modulo ppp is a quadratic nonresidue modulo ppp.
  • (iii) The product of two quadratic nonresidues modulo ppp is a quadratic residue modulo ppp.

Proof. This proof uses Fermat's Little Theorem which states that ap11(modp)a^{p-1} \equiv 1 \pmod pap11(modp) for prime ppp and for integers aaa such that ppp does not divide aaa.

Let gFpg \in \mathbb{F}_p^*gFp be a generator for Fp\mathbb{F}_p^*Fp. We claim that even powers of ggg are quadratic residues modulo ppp. This claim can be proved with a proof by contradiction. Suppose that g2k+1g^{2k+1}g2k+1 is a quadratic residue modulo ppp. Then

g2k+1m2(modp)g^{2k+1} \equiv m^2 \pmod pg2k+1m2(modp)

for some integer mmm. From Fermat's Little Theorem, we know that mp11(modp)m^{p-1} \equiv 1 \pmod pmp11(modp). So

mp1(m2)p12(g2k+1)p12gk(p1)gp12(gp1)kgp12gp12(modp)m^{p-1} \equiv (m^2)^{\frac{p-1}{2}} \equiv (g^{2k+1})^{\frac{p-1}{2}} \equiv g^{k(p-1)} \cdot g^{\frac{p-1}{2}} \equiv (g^{p-1})^k \cdot g^{\frac{p-1}{2}} \equiv g^{\frac{p-1}{2}} \pmod pmp1(m2)2p1(g2k+1)2p1gk(p1)g2p1(gp1)kg2p1g2p1(modp)

which implies gp121(modp)g^{\frac{p-1}{2}} \equiv 1 \pmod pg2p11(modp). But this contradicts the fact that ggg is a generator for Fp\mathbb{F}_p^*Fp as there can only be one value of lll with 0l<p10 \leq l < p-10l<p1 such that gl1(modp)g^l \equiv 1 \pmod pgl1(modp) (that value is 000).

Next, we let aaa and bbb be quadratic residues modulo ppp, and let ccc and ddd be quadratic nonresidues modulo ppp. We can write

ax2(modp)by2(modp)cw2j+1(modp)dz2k+1(modp)\begin{aligned} a &\equiv x^2 \pmod p \\ b &\equiv y^2 \pmod p \\ c &\equiv w^{2j+1} \pmod p \\ d &\equiv z^{2k+1} \pmod p \end{aligned}abcdx2(modp)y2(modp)w2j+1(modp)z2k+1(modp)

To prove (i), we see that

abx2y2(xy)2(modp)ab \equiv x^2 y^2 \equiv (xy)^2 \pmod pabx2y2(xy)2(modp)

and so ababab is a quadratic residue modulo ppp.

To prove (ii), we see that

acx2w2j+1(modp)ac \equiv x^2w^{2j+1} \pmod pacx2w2j+1(modp)

which cannot be expressed to an even power, hence, is a quadratic nonresidue modulo ppp.

Tp prove (iii), we see that

cdw2j+1z2k+1wz2j+2k+2(wzj+k+1)2(modp)cd \equiv w^{2j+1}z^{2k+1} \equiv wz^{2j + 2k + 2} \equiv (wz^{j+k+1})^2 \pmod pcdw2j+1z2k+1wz2j+2k+2(wzj+k+1)2(modp)

and so cdcdcd is a quadratic residue modulo ppp.

Solving the challenge

The findQNR function uses Euler's Criterion to find a quadratic nonresidue modulo ppp. The findQR function returns a quadratic residue modulo ppp. We see that s0s_0s0 is a quadratic nonresidue modulo ppp and s1s_1s1 is a quadratic residue modulo ppp. Our goal has become to determine whether or not the plaintext value being encrypted was a quadratic nonresidue or a quadratic residue. We are given c1gr(modp)c_1 \equiv g^r \pmod pc1gr(modp) and c2myr(modp)c_2 \equiv my^r \pmod pc2myr(modp) and yyy and ppp. Is there a way to determine whether mmm is a quadratic residue or not from these values?

Notice that if yyy is a quadratic residue, then c2c_2c2 is a quadratic residue if and only if mmm is a quadratic residue.

Else, if yyy is a quadratic nonresidue, whether or not c2c_2c2 is a quadratic residue will depend on c1c_1c1 (this is beacuse yrgrx(modp)y^r \equiv g^{rx} \pmod pyrgrx(modp)). If c1c_1c1 is a quadratic residue, then c2c_2c2 is a quadratic residue if and only if mmm is a quadratic residue. Else, if c1c_1c1 is a quadratic nonresidue, then c2c_2c2 is a quadratic residue if and only if mmm is a quadratic nonresidue.

In summary:

If y is a QR:
    c2 is a QR iff M is a QR

If y is a QNR:
    if c1 is a QR:
        c2 is a QR iff M is a QR
    if c1 is a QNR:
        c2 is a QR iff M is a QNR

We can now write our exploit script:

from pwn import remote

def ec(x, p):
    q = (p - 1) / 2
    return pow(x, q, p)

def qr(x, p):
    return ec(x, p) == 1

conn = remote('guess.q.2020.volgactf.ru', 7777)

y, p = map(int, conn.recvline().split('= (')[1][:-3].split(', '))

for i in range(1000):
    c1, c2 = eval(conn.recvline())

    b = ''
    if qr(y, p):
        if qr(c2, p):
            b = '1'
        else:
            b = '0'
    else:
        if qr(c1, p):
            if qr(c2, p):
                b = '1'
            else:
                b = '0'
        else:
            if qr(c2, p):
                b = '0'
            else:
                b = '1'
    conn.sendline(b)
    print('challenge', i)
print(conn.recvline())
